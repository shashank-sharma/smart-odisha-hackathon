API Documentation

/api/chart/complaint
To get all the complaint available in format: complaints count and subject list

Available Filters:

gender :- Gender can be ['male', 'female', 'others']

state :- State can be values defined in shortforms: Madhya Pradesh = MP

city :- City needs to be specific word (BETA)

from :- If you want results from specific date till present then use from
/api/chart/complaint?from=2018-01-23
This compiles to complaints from 23rd of Jan, 2018 till present. So your format must be yyyy-mm-dd strictly else no result will be there.

to :- To specify the end date. Means from past to that date it will fetch all complaints

Note: You can use both from and to parameters to get specific range of complaints

/api/chart/complaint?from=2018-01-10&to=2018-01-15
This will get complaints from 10th Jan to 15th Jan of 2018

tags :- Specify tags from available list to get complaints according to that. (BETA)

week :- It can be either true or false. Week will get complaints from past 7 days of data

Note: This should not be used with from and to parameters

date :- Get complaint of specific date by mentioning it in format yyyy-mm-dd

resolved :- Get complaint which are resolved. Value can be either true or false



/api/chart/tags
It helps you get all the trending tags available. Format: "tags": Showing all tags with count, "top": Showing top trending tag
This api call needs no parameter

api/chart/week
This on is little similar to our first API url but with single call you will get result of all complaints from past 7 days.
This will give you total complaints, data where data contains values of each date. Each date data will have male, female, others count.
